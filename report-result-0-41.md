# Отчёт по работе [result-0-41.ipynb](result-0-41.ipynb)

## 1. Постановка задачи

Цель работы — построить регрессионную модель, прогнозирующую оценку студента (`mark`) по дисциплине на основе данных об учебном процессе и личных данных студента. Датасет — `onti-students-performance` (соревнование по успеваемости студентов). Метрики — `R²`, `RMSE`, `MAE`, `MedAE`, `MAPE`.

## 2. Исходные данные

- [X_train.csv](onti-students-performance/X_train.csv) — обучающие признаки, **105 058** записей, 7 колонок: `STD_ID, НАПРАВЛЕНИЕ, ГОД, КУРС, СЕМЕСТР, АТТЕСТАЦИЯ, ДИСЦИПЛИНА`.
- [studs_info.csv](onti-students-performance/studs_info.csv) — справочник студентов, **5 466** строк, 13 колонок (пол, статус, форма обучения, образование и т.д.).
- [y_train.csv](onti-students-performance/y_train.csv) — целевая переменная `mark` (оценка 0–5).
- [X_test.csv](onti-students-performance/X_test.csv) — **26 265** строк для предсказания.

В исходниках встретились мусорные имена колонок (`'   '`, `'   number'`), переименованы в `number1`/`number2`.

## 3. Предобработка данных

### 3.1. Очистка `studs_info`
- Удалены неинформативные/дублирующие столбцы: `Дата выпуска` (5002 пропуска из 5466), `Шифр`, `направление (специальность)`, `Дата выдачи`.
- В справочнике обнаружены **дубликаты по `STD_ID`** (один студент мог иметь несколько записей с разными школами/номерами). Применён `drop_duplicates(['STD_ID'])`.
- Пропуски заполнены: числовые — средним; категориальные — модой.

### 3.2. Объединение
`X_train` объединён с `studs_info` через `LEFT JOIN` по `STD_ID`. Итог — **105 058 × 15** без потери строк.

## 4. Разведочный анализ (EDA)

Построены круговые диаграммы по категориальным признакам (`Пол`, `Статус`, `Категория обучения`, `Форма обучения`, `Образование`, `Что именно закончил`, `НАПРАВЛЕНИЕ`, `ГОД`, `АТТЕСТАЦИЯ`).

Особые наблюдения:
- В `ДИСЦИПЛИНА` — **1 400 уникальных значений**, длинный хвост.
- В `STD_ID` — 5 084 уникальных студента, до 61 записи на студента.
- Корреляционная матрица (heatmap) построена после кодирования через `cat.codes`. Сильных линейных корреляций между признаками нет, ожидаемая связь между `КУРС`/`СЕМЕСТР` подтверждается.

## 5. Подготовка матрицы признаков

В финальной версии:
- **Категориальные признаки** кодируются через `OrdinalEncoder` с `handle_unknown='use_encoded_value', unknown_value=-1`. Энкодер обучается **один раз** на train и применяется к valid/test с теми же кодами (устранена несогласованность кодов, которая была при `cat.codes`).
- **Удалены** `STD_ID`, `number1`, `number2` — это псевдо-идентификаторы студента, которые порождали leakage.
- **Добавлены 4 history-фичи** через **out-of-fold target encoding**:
  - `stud_mean_grade` — средняя оценка студента (OOF на train, полные средние на valid/test).
  - `stud_count` — количество записей студента в train.
  - `discipline_mean_grade` — средняя оценка по дисциплине.
  - `direction_mean_grade` — средняя оценка по направлению.

Постобработка предсказаний — `clip([0, 5])`.

## 6. Базовое сравнение моделей

Обучены пять регрессоров с дефолтными параметрами (на финальной матрице признаков с history-фичами):

| Модель | RMSE | MAE | MedAE | MAPE | **R²** |
|---|---|---|---|---|---|
| LinearRegression | 0.5565 | 0.4491 | 0.3981 | 0.1141 | 0.4570 |
| SGDRegressor | 1.1088 | 0.8118 | 1.0000 | 0.2378 | −1.1554 |
| GradientBoostingRegressor | 0.5469 | 0.4445 | 0.3908 | 0.1128 | 0.4757 |
| **RandomForestRegressor** | **0.5348** | **0.4213** | **0.3500** | **0.1067** | **0.4985** |
| MLPRegressor | 0.5694 | 0.4660 | 0.4183 | 0.1167 | 0.4317 |

**RandomForest снова лучший** — R²=0.4985.

`SGDRegressor` остался плохим (R²=−1.16) из-за отсутствия масштабирования числовых признаков, но без катастрофы прежних версий, где R² ≈ −8.4·10³⁶.

## 7. Тюнинг RandomForest

`RandomizedSearchCV(n_iter=20, cv=3, scoring='r2')` по сетке:
```python
n_estimators ∈ {200, 400, 600}
max_depth ∈ {None, 15, 20, 30}
min_samples_leaf ∈ {1, 3, 5}
min_samples_split ∈ {2, 5, 10}
max_features ∈ {sqrt, 0.5, 0.7}
```

**Результат**:
- Лучшие параметры: `n_estimators=600, max_depth=20, min_samples_leaf=1, min_samples_split=2, max_features='sqrt'`
- **Лучший CV R²: 0.5106**

Метрики на holdout (10% от train):

| Метрика | Значение |
|---|---|
| **R²** | **0.5249** |
| RMSE | 0.5206 |
| MAE | 0.4138 |
| MedAE | 0.3450 |
| MAPE | 0.1048 |

## 8. Анализ warm/cold-start

В holdout 10 506 записей. Из них:

| Подмножество | Записей | R² |
|---|---|---|
| Warm-start (студент известен модели) | 10 503 (100%) | **0.525** |
| Cold-start (новый студент) | 3 (~0%) | 0.0 (слишком мало для достоверной метрики) |

**В тестовой выборке (`X_test.csv`) ситуация аналогична**:
- Известных студентов: **26 253 из 26 265 (99.95%)**.

То есть данные соревнования построены так, что **тестируется warm-start**: те же студенты появляются в train и test с разными дисциплинами/семестрами. Это объясняет, почему history-фичи дают такой большой буст.

## 9. История изменений R² по версиям

| Версия | Подход | Holdout R² | Комментарий |
|---|---|---|---|
| v1 (исходник) | `cat.codes` + random split + STD_ID/number1/number2 в признаках | 0.46 | Завышено leakage'ом по группе |
| v2 (после fix) | `OrdinalEncoder` + GroupShuffleSplit + без STD_ID | 0.225 | Честная цифра для cold-start, но не отражает реальный тест |
| **v3 (финал)** | **+ OOF target encoding (history-фичи) + обычный split** | **0.5249** | Реалистичный сценарий, соответствующий структуре X_test |

## 11. Streamlit-UI ([app.py](app.py))

Интерактивный интерфейс для предсказания оценки студента:

- **Выбор студента** из справочника (warm-start) или ручной ввод (cold-start).
- При выборе известного студента UI отображает его реальную среднюю оценку и количество записей в истории, помечает режим как **✅ warm-start**.
- Для нового студента — **❄️ cold-start**, history-фичи равны глобальному среднему.
- На инференсе модель автоматически собирает 4 history-фичи через lookup-таблицы из `meta.pkl`.
- Выводит: дробное предсказание (0–5), округлённую оценку с подписью (удовл/хор/отл), progress bar.

Запуск:
```bash
source .venv/bin/activate
streamlit run app.py
```

## 12. Ключевые методологические улучшения

| Проблема в первой версии | Решение |
|---|---|
| `cat.codes` независимо на train/test → разные коды для одной категории | `OrdinalEncoder` с `handle_unknown` обучается один раз на train |
| `STD_ID`/`number1`/`number2` в признаках → leakage | Удалены, заменены на target-encoded history-фичи |
| Тюнили GBR, хотя лучше был RF | Тюнинг **именно** RF через `RandomizedSearchCV` |
| Нет history-фич → модель не понимает, какой студент сильный | OOF target encoding по `STD_ID`, `ДИСЦИПЛИНА`, `НАПРАВЛЕНИЕ` |
| Нет UI | Streamlit с двумя режимами (warm/cold) |
| Нет анализа warm/cold | В отчёте выводятся отдельные метрики по подмножествам holdout |

## 13. Вывод

Построен пайплайн прогнозирования оценки студента, состоящий из:
1. Очистки и объединения данных `X_train.csv` со справочником `studs_info.csv`.
2. EDA с круговыми диаграммами категориальных признаков и матрицей корреляции.
3. Сравнения 5 регрессоров на честно подготовленной матрице признаков с OOF target encoding.
4. Тюнинга лучшей модели (RandomForest) через RandomizedSearchCV.
5. Анализа warm/cold-start сценариев.
6. Сохранения артефактов и Streamlit-UI для интерактивной работы.

**Финальное качество модели: R² = 0.5249, MAE = 0.4138 на отложенном holdout.** Это в 2.3 раза выше честной cold-start метрики (0.225) и соответствует реальному распределению тестовых данных, где 99.95% студентов уже есть в train.

Главный методологический вклад — **разделение признака "конкретный студент" (псевдо-ID, leakage) на содержательные target-encoded фичи** (средние оценки), что позволяет модели использовать историю корректно, без утечки таргета.
